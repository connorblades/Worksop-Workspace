# Key Holder and Out-of-Hours Log

**Worksop Workspace** · Operations · Facilities · *Phase 1 reference*

Status: **Draft v1 — needs Connor review.** Last updated: 16 May 2026.

---

This document is two things:

1. The **register** of who holds keys (or door codes / fobs) to Worksop Workspace.
2. The **log** of out-of-hours access (anyone in the building outside normal published hours).

Both exist for security, audit, and to support insurance and alarm-system response. If something happens out-of-hours, this is the first thing the police, fire service, or insurer will ask about.

## 2. Out-of-hours access log

Anyone in the building outside the published opening hours signs in here. Includes:

- Connor or any staff member arriving early / staying late
- The landlord visiting
- Members with private offices using out-of-hours access (when offered)
- Cleaners, contractors, fire-system engineers, network engineers, alarm responders
- Anyone called in for an out-of-hours emergency

| Date | Name | Reason | Time in | Time out | Areas accessed | Signed |
|---|---|---|---|---|---|---|
| | | | | | | |

The log lives in two forms:

- **Paper:** a bound notebook at the reception desk, used as the primary record. Bound because loose-leaf logs are too easy to tamper with retrospectively.
- **Digital backup:** a Google Sheet that Connor updates from photos of the paper log weekly. Sheet is at `Worksop Workspace HQ > 02-operations > out-of-hours-log`.

## 4. Procedure when a key holder leaves

When a key holder no longer needs access (staff leaves, contract ends, key returned):

1. Physical key returned to Connor on the day.
2. Door code rotated within 24 hours; new code issued only to current key holders.
3. Alarm monitoring company notified of the removal within 24 hours.
4. This document updated: date revoked filled in, row not deleted (history preserved).
5. CCTV: a clip of the key being returned (or of the door code being demonstrated as no-longer-working) is preserved for 60 days.

---

## 5. Procedure on a lost or compromised key

1. Reported to Connor immediately.
2. Door codes changed within 24 hours.
3. Physical locks recoded as soon as a locksmith can attend (within 7 days at most).
4. Incident logged in the [Accident and Incident Report](../health-and-safety/accident-and-incident-report.md) (yes, the same book — security incidents are logged in there too).
5. Insurer notified if the loss may give rise to a claim.
