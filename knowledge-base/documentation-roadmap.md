## 4. What lives in Drive, not the repo

Per the [operating manual](./operations/operating-manual.md#9-issues-and-emergencies),
sensitive operational data lives in the private Drive at
`Worksop Workspace HQ` rather than this public repo. That includes:

- Supplier rates and account numbers
- Utility provider references and meter numbers
- IT credentials (Wi-Fi admin, router login, member-app admin)
- CCTV provider portal access
- Stripe, GHL, Google Workspace admin
- Specific insurance policy numbers
- Bank details and direct debit references
- Lease and landlord correspondence
- Any signed contracts with members

This document maps the **structure** of those Drive docs and the
process around them — populate the sensitive specifics in Drive.

## 5. How to maintain this

- **Touch this file when you add or retire a doc.** It's the index;
  it goes stale fastest.
- **Status discipline.** Every doc in `knowledge-base/` carries a
  `Status:` line. Update it whenever the doc materially changes.
- **Source of truth.** The `.md` is canonical. PDFs in
  `Worksop Workspace HQ / 02-operations / _pdfs/` are derivatives —
  regenerated by `tools/pdf-generator/`.
- **Cross-link liberally.** Inline references between docs are how a
  small team navigates without a search bar. See the operating
  manual for the linking pattern.
